## Abstract

You can build anything with AI. The open question is whether that output reflects durable skill.
Skillcraft addresses this by turning development activity into a verifiable evidence chain that can be checked
with public rules and public artifacts.

The design uses existing repositories as trust boundaries and git as the record of truth.
A developer enables Skillcraft, captures skill usage in local events, generates proof objects on commit,
and submits claims for credentials that are then validated against policy.
The result is a practical system for credible capability signals that does not depend on a hidden telemetry backend.

This whitepaper documents:
- the ecosystem architecture,
- evidence and claim data flow,
- security and trust assumptions,
- current implementation status,
- and near-term roadmap to fully operationalize public loadout and route surfaces.
