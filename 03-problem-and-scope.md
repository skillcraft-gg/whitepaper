## Problem and Scope

Developers and platform builders face a recurring gap:
projects are visible, but capability proofs are not.
A claim such as "I can design secure systems" has no shared, machine-checkable basis unless the repository history itself
contains the supporting evidence.

Skillcraft defines a strict scope:
- not a reputation game,
- not a full identity platform,
- not a replacement for manual review.

Instead, Skillcraft is a verifiable signal layer around normal git workflows.
The system records what was done, where it was done, and under what skill or loadout context.

The practical problem statement becomes:

*How can we preserve development velocity while increasing signal quality?*

Skillcraft answers with:
- structured evidence anchored to commits,
- deterministic registry objects,
- and policy-based issuance.

The core claim is that this is good enough for public trust decisions when each step is bounded and auditable.
